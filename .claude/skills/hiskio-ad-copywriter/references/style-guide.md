# HiSKIO 廣告文案語氣風格指南

本指南從 12 個現有文案範例（vibe coding1-7、claude1-3、rag1-2）萃取，作為生成新文案的風格基準。

---

## 一、品牌語氣

**核心定位**：AI 時代的實戰課程平台，幫助「普通人」真正用 AI 創造價值。

**語氣關鍵字**：
- 輕鬆但有料（不裝學術、不過度正式）
- 數據導向（具體數字比抽象形容有說服力）
- AI 時代定位（強調「現在正在發生」，而非「未來可能發生」）
- 接地氣（用「你」直接對話，不用「讀者們」）

**語氣範例對比**：
- ❌ 「本課程將全面提升學員的 AI 應用能力」
- ✅ 「學完你就能用 AI 做出以前要三個月才能完成的東西」

## 二、開頭 Hook 的常見模式

### 模式 1：時事破題
以最近發生的外部事件直接開頭，製造話題感。

> 「【Instagram 執行長內部信件：2026 終結混合辦公】」
> 「AI 開始能操作你的電腦了——但你準備好了嗎？」

**特徵**：標題感強、有時間感、引發好奇

### 模式 2：反差數字
用一個讓人意外的數字或對比製造張力。

> 「每個月，我自己親手寫的程式碼以 5% 的速度遞減。不到一年，我幾乎只修改最後提交程式碼的 15%」
> 「一個人，加上 AI，24 小時——從零開始⋯⋯做完整套技術底層。」

**特徵**：具體、可信、讓人想知道「怎麼辦到的」

### 模式 3：問句切入
用一個讀者心中的疑問或擔憂開場。

> 「『用 AI 寫程式，能做出真正的東西嗎？』這大概是過去一年，大家對 Vibe Coding 最大的疑慮。」

**特徵**：直接觸碰讀者的內心獨白，讓人感覺「這是在跟我說話」

### 模式 4：故事切入
用一個真實人物或情境的故事開場。

> 「新媒體藝術家吳哲宇說，他在威尼斯雙年展上被一位義大利策展人問了這個問題⋯⋯」

**特徵**：有人物、有場景、有衝突，讀者容易代入

## 三、文案結構規律

大多數文案遵循以下結構（不一定全部出現，但順序不變）：

```
① 開頭 Hook（時事 / 數字 / 問句 / 故事）
↓
② 問題 / 現象陳述（讀者遇到的困境或趨勢）
↓
③ 轉折（但是⋯⋯ / 然而⋯⋯ / 現在情況不一樣了）
↓
④ 解決方案 / 課程價值（學了能做什麼）
↓
⑤ 課程介紹（課程名稱、學習成果、✅ 列點）
↓
⑥ CTA（立即報名 / 場次資訊 / 連結）
```

## 四、常見 CTA 格式

**體驗課（免費直播）標準格式**：
```
《[課程名稱]（體驗課）》免費名額倒數

場次資訊：
日期：MM 月 DD 日
時間：19:00～20:30
地點：線上直播
立即報名：[連結]
```

**募資 / 正式課程格式**：
```
[課程名稱] 募資優惠中
>> [連結]
```

**粉專貼文結尾**（較輕鬆）：
```
打造 AI 時代百倍產能的工程師工作流，從這堂課開始！
>> [連結]
```

## 五、學習成果列點格式

使用 ✅ 符號，句子控制在一行內，動詞開頭：

```
✅ [動詞] + [具體成果]（不要抽象）
✅ [動詞] + [具體成果]
✅ [動詞] + [具體成果]
```

**範例**：
- ✅ 打造可實際運作的 AI 搜尋助理
- ✅ 建立可量化、可重複的優化 Pipeline
- ✅ 導入使用者回饋，讓搜尋越用越準

## 六、禁止 / 注意事項

### 事實與引用
- ❌ 不能宣稱某人「不會寫程式」（除非本人自述）
- ❌ 引用某人的話必須有原文依據（當事人 FB、訪談原文）
- ❌ 不能宣稱學了課程「保證薪資提升 X%」等無法核實的結果
- ⚠️ 媒體報導的轉述 ≠ 當事人自述，使用前需區分

### 語氣與風格
- ❌ 不誇大宣稱（「最強」、「唯一」、「保證」）
- ❌ 不過度催促（「最後機會！」、「再不買就來不及了！」）
- ❌ 不使用大量感嘆號
- ❌ emoji 不要超過 3 個（除非品牌風格如此）

### 技術術語
- ⚠️ 針對陌生受眾，技術術語需要白話解釋
- ✅ 可以出現術語，但後面要加括號說明（例：RAG（讓 AI 能查資料的技術））

## 七、各課程的核心訊息

### Vibe Coding
- 核心：AI 時代，想法比技術重要，AI 讓普通人也能把想法做出來
- 受眾：不會寫程式的非技術人員、想轉型的職場人
- 關鍵數字：10,000+ 同學報名（截至 2025 年 Q1）

### Claude Code
- 核心：工程師用 AI 工具重建工作流，產出比以前高出數倍
- 受眾：有程式基礎的工程師、想提升效率的開發者
- 老師角色：亞馬遜工程師，有第一手 AI 轉型經歷

### RAG 精準搜尋實戰
- 核心：RAG 不只是串 API，讓搜尋穩定、精準、越用越準才是關鍵
- 受眾：後端工程師、需要做公司 AI 功能的開發者
- 老師：張維元，資料科學家背景，有實戰講師經驗
